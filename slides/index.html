Le client

Dans le cas d'application web, le client est le navigateur web (chrome, firefox, opera ...) fonctionnant sur n'importe quel device


  1. Préparer une requête HTTP suivant la demande du client, par exemple accéder à la page web https://github.com/Mcdostone/HTML-CSS
  2. Envoyer la requête au serveur
  3. Attendre la réponse du serveur
  4. Afficher la réponse du serveur dans l'interface du client

Une chose à comprendre coté client ! Une seule !

  • Le client ne sait interpréter que du contenu HTML, CSS et JS !
  • Voir la formation dédiée
  • Ne faites jamais cette erreur ! JAMAIS

Le serveur

Machine connectée au réseau. Ici nous nous occupons uniquement du serveur web

  1. Attendre une requête HTTP d'un client
  2. Préparer la réponse suivant la demande du client
  3. Envoyer la réponse au client

Regroupe trois parties:

Le back-end

Configuration du serveur

Installer tous les programmes importants au bout fonctionnement de notre solution informatique

  • Installation d'un serveur web (Apache, nginx, rails, nodejs...)
  • Installation d'un SGBD (MySQL, mariaDB, PostgreSQL)
  • Installation des librairies
  • Configuration du serveur en lui-même (adresse IP, ports, sécurité ..., connexion au SGBD)
